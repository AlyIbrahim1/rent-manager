## Design Context

### Users
Property managers and landlords working primarily from a desk in a professional office context. They oversee multiple units, track rent collection, monitor overdue balances, and need confidence at a glance without feeling buried in admin software.

### Use Cases
- Review portfolio health quickly at the start of the day.
- Identify overdue renters and take the next action fast.
- Add renters, record payments, and inspect payment history without losing page context.
- Move through the product with the feeling of a curated financial workspace rather than a cramped CRUD tool.

### Brand Personality
**The Architectural Editor**

Three words:
- structured
- precise
- elevated

The product should feel editorial, calm, and authoritative. It is not playful, loud, glassy, neon, or overtly “techy.” It should suggest premium wealth-management software adapted for rental operations.

### Quality Bar
**Exact design-system execution**

This project should not merely feel “on brand.” It should follow the Ledger design system and dashboard UI kit as literally as possible. When a kit component exists, future work should match that component’s structure, spacing, surface treatment, typography, and control placement instead of reinterpreting it.

### Source Of Truth
Use these in order:
1. `.agents/skills/design-system/ui_kits/dashboard/index.html`
2. `.agents/skills/design-system/ui_kits/dashboard/*.jsx`
3. `.agents/skills/design-system/colors_and_type.css`
4. `.agents/skills/design-system/README.md`

If a live product surface differs from the source kit, the kit wins unless product behavior makes an exact match impossible.

### Performance Constraints
- Motion must feel smooth on typical laptops and mid-range phones.
- Prefer `transform` and `opacity` for animation.
- Floating UI must remain responsive and never block interaction longer than necessary.
- Respect `prefers-reduced-motion` everywhere.

## Creative Direction

### Aesthetic Direction
**The Curated Ledger**

The interface should read like a stack of refined paper layers on a light desk: soft tonal shifts, careful spacing, strong typography, and selective emphasis. The experience should feel composed and expensive without becoming ornamental.

### Theme
Light mode by default. This product is used in daylight, office, and administrative work contexts where clarity and scanability matter more than mood lighting.

### Accent Strategy
Emerald is reserved for high-signal success and payment actions. It must stay rare. Do not use it decoratively in menus, section headers, or passive chrome.

## System Principles

### 1. UI Kit Parity Over Interpretation
When the dashboard UI kit provides a component pattern, follow it directly. Do not “upgrade,” “modernize,” or improvise around it unless there is a concrete product constraint.

### 2. No-Line Rule
Avoid 1px borders for sectioning. Use tonal shifts, spacing, and shadow weight instead.

### 3. Floating Surfaces Must Feel Solid
Menus, popovers, drawers, and modals should always use full-opacity surfaces with meaningful elevation. No transparent shells, no glassmorphism, no decorative glow.

### 4. Architectural Geometry
Favor `rounded-sm` and `rounded-md` over overly soft corners. Pills are reserved for chips and status badges. Most structural UI should feel crisp and squared.

### 5. Typography Carries Authority
Use Manrope for display and editorial emphasis. Use Inter for operational UI and dense body copy. Hierarchy should come from contrast, not decoration.

### 6. Soft Lift Over Hard Chrome
Depth should come from stacked tonal planes and diffused shadows. Interfaces should feel layered, not outlined.

### 7. High-Signal Copy
Short labels, restrained metadata, and no redundant explanatory filler. Copy should feel confident and trimmed.

## Typography Rules

### Fonts
- Display and headings: `Manrope` 700
- Body, labels, inputs, operational copy: `Inter` 400/500/600/700

### Scale
- Large KPI figures: `3.3rem` to `3.5rem`
- Page title: `2.25rem` to `2.5rem`
- Modal title: around `1.75rem`
- Section titles: `1.15rem` to `1.4rem`
- Body copy: `14px`
- Metadata labels: `11px` to `12px`, uppercase, `0.14em` to `0.20em` tracking

### Numeric Data
All important financial figures should use tabular numerals and `toLocaleString()`.

## Layout Rules

### Overall Structure
- Desktop dashboard uses a fixed left sidebar and a sticky top header.
- Main content follows the UI kit sequence:
  `Portfolio Overview` -> `Renters Ledger` + `Renter Detail Panel` -> supporting secondary surface
- Main desktop width should feel like the 1280px UI kit layout, not a loose fluid marketing page.

### Spacing
- Major section gaps: `24px` to `32px`
- Card gaps: `12px` to `16px`
- Dashboard sidebar width should stay aligned to the kit’s fixed rail treatment.

### Background
Use the Ledger radial background composition, not a flat fill and not a dark canvas.

## Surface Inventory

### Auth
- Login should follow the UI kit auth screen: centered translucent card, logo lockup, compact form stack, gradient submit button, restrained supporting controls.
- Sign-up and forgot-password states should stay inside the same auth shell language instead of becoming separate visual systems.

### Sidebar
- Sidebar is a full-height light surface with logo, small uppercase descriptor, stacked nav items, and the primary `Add Renter` CTA at the bottom.
- Non-active nav items use `Soon` labels.

### Header
- Header contains search, notifications, and profile access.
- Do not duplicate the primary `Add Renter` CTA in the header.
- Controls should feel compact, tonal, and balanced.

### Portfolio Overview
- Follow the kit’s KPI composition closely.
- Use large editorial numbers, short labels, and a small set of actions.
- The `Receive Payment` action can use emerald.

### Renters Ledger
- Follow the kit ledger-card pattern closely: apt badge, overdue pill, renter name, nested rent summary panel.
- Keep card spacing, tonal nesting, and hover lift restrained.

### Renter Detail Panel
- Match the kit right-panel behavior and visual hierarchy.
- Show a strong empty state when no renter is selected.
- Payment history should feel quiet and informational, not table-heavy.

### Modals
- Modal shell must follow the kit pattern:
  tonal header + white section cards + tonal footer
- Outer radius should feel elevated and premium.
- Inputs use filled gray surfaces with no hard border.
- Use the navy gradient for standard primary actions.

### Menus And Popovers
- Use solid white shells with tonal internal zoning.
- Prefer diffused elevation and quiet hover states.
- Avoid bright blue or generic SaaS dropdown styling.

## Component Guidance

### Buttons
- Primary actions use the navy gradient.
- Secondary actions use tonal surfaces.
- Tertiary high-signal financial actions use dark emerald with mint text.
- Hover states should feel like a soft lift: subtle scale, slight tonal shift, restrained shadow.

### Inputs
- Default input fill: `surface highest`
- Focus state: lighter surface with a soft primary ring
- Labels stay uppercase and tightly tracked

### Chips And Badges
- Unit badges are compact rectangular labels.
- Status chips are rounded pills.
- `Scaffolded` and `Soon` use muted tonal chips, never loud accents.

### Profile Menu
- Treat it as a floating ledger card, not a generic account dropdown.
- Use a tonal introduction zone and a quieter action zone.
- The primary identity marker should feel deliberate and architectural, not playful.

## Motion Direction
- Use smooth deceleration: quart, quint, expo.
- Motion should clarify hierarchy and state changes, not call attention to itself.
- Entrances can feel composed and editorial.
- Exits should be slightly faster than entrances.
- Avoid bounce, elastic, or playful motion language.

## Anti-References
- No glowing dark-mode SaaS styling.
- No generic avatar-menu blue treatment.
- No line-heavy enterprise admin look.
- No decorative gradients outside weighted primary CTAs.
- No stacked mini-cards inside already-floating UI unless the hierarchy truly needs it.
- No “brand-inspired” redesigns that drift from the Ledger dashboard kit layout.
